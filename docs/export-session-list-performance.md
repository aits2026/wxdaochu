# 导出数据会话列表性能优化清单

最后更新: 2026-02-23

## 背景

导出数据页的会话选择面板在以下场景存在卡顿感:

- 首次进入页面后，消息数量统计返回时列表重排
- 搜索输入时会话过滤与排序频繁触发
- 会话数量较多时，列表全量渲染与重绘开销较大
- 后端批量统计消息数量时，表名匹配逻辑存在不必要重复计算

## 优化项总览

### 阶段 1（本次执行，低风险高收益）

- [x] `DOC-1` 建立性能优化文档与执行记录
- [x] `FE-1` 会话分类数量改为单次遍历统计（避免 render 中重复 `filter`)
- [x] `FE-2` 会话过滤/排序改为派生计算（`useMemo`），移除额外 `filteredSessions` 状态更新
- [x] `FE-3` 搜索输入使用延迟值（`useDeferredValue`），降低输入时重算频率
- [x] `FE-4` 消息数量返回时使用 `startTransition` 更新，降低重排对交互的阻塞感
- [x] `FE-5` 会话列表改为虚拟列表渲染（`react-window`）
- [x] `FE-6` 选中会话数据改为映射查找（避免 render 期重复线性查找）
- [x] `BE-1` 优化 `getSessionMessageCounts` 的 hash 匹配逻辑（去掉重复 `Object.keys().find()`）

### 阶段 2（后续优化候选）

- [x] `BE-2` 为会话消息数统计增加缓存（按数据库文件变更/mtime 失效）
- [x] `BE-3` 将大批量消息数统计放到独立 worker / 子进程，减少主进程阻塞（已实现 worker 优先 + 主线程回退）
- [x] `BE-4` 按当前筛选分类优先统计，其他会话延迟统计（已实现分类优先；可视范围优先留作后续增强）
- [ ] `FE-7` 导出会话列表头像懒加载（降低首屏图片解码/请求开销）
- [ ] `FE-8` 增加“排序方式”切换（最近聊天 / 消息数），默认最近聊天以减少列表跳动

### 本次变更的预期行为变化

- 首次进入导出页时，会话列表先按原始顺序显示，消息数量统计完成后再以较低优先级重排（私聊/群聊）
- 搜索输入时卡顿感应明显下降（过滤与排序使用延迟值）
- 会话数量较多时滚动与切换分类更顺滑（虚拟列表仅渲染可视区域）
- 重复进入导出页/刷新且数据库未变化时，消息数量统计可直接命中缓存，减少全量 `COUNT(*)` 扫描
- 首次进入导出页时，当前分类（默认私聊）会优先完成消息数统计与排序，其他分类延后补齐，首屏可用更快
- 大批量会话消息数统计时，主进程阻塞感进一步下降（worker 执行统计；失败时自动回退）
